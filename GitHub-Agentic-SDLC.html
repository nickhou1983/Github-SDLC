Issue AI — 需求管理的 AI 入口

把 Issue 从「人工文本」变成「AI 可执行的任务定义」

是什么

GitHub Issues 内置的 Copilot 能力集。自然语言一句话即可生成结构化 Issue（标题、复现步骤、标签），支持批量创建、模板匹配与重复检测。

Issue 同时是 Coding Agent 的最佳提示词载体：标题 = 目标，正文 = 验收标准，评论 = 补充上下文。

自然语言创建

口述需求 → 自动结构化 Issue，套用仓库的 Issue Template / Forms，支持一次批量创建多个

摘要与解释

长讨论串一键摘要；解释 Issue 中引用的代码与报错，快速建立上下文

子任务拆解

大需求拆成 Sub-issues 树，每个子任务可独立分配给人或 Copilot

assign 给 Copilot

Issue 直接指派给虚拟协作者 Copilot，即刻触发 Coding Agent 开始工作

最佳实践：写清「验收标准」是 AI 交付成功率的第一杠杆；适合委托 Bug 修复 / 测试补全 / 文档更新，不适合目标模糊的需求。

Coding Agent — 云端异步自主编码

Issue 进，Draft PR 出 —— 端到端 AI 交付的核心载体

是什么

GitHub 官方云端编码智能体。把 Issue 或一句话任务交给它，在 GitHub Actions 驱动的临时沙箱中：分析代码库 → 制定计划 → 编码 → 跑测试 → 自我修复，最终产出 Draft PR 等你评审。

入口：Issue assign、Agents 面板、VS Code、CLI、Mobile。

沙箱 + 防火墙

临时环境用完即毁；默认限制出站网络防数据外泄，管理员可配白名单

权限边界

只能推 copilot/* 分支；不能 approve / merge 自己的 PR；触发的 CI 需人批准

MCP 可扩展

通过 Model Context Protocol 接入浏览器测试、内部 API、数据库 schema 等外部工具

全程可审计

完整 Session Log：每一步推理、命令、文件变更都可回放审查

适用场景：Bug 修复 / 测试覆盖 / 技术债 ★★★★★；中型功能 ★★★；跨服务架构变更与高风险模块仍需人主导。

Code Review Agent — AI 代码评审

AI 负责广度（每行都看），人负责深度（业务与架构）

是什么

Copilot 作为「第一位评审者」自动审查 PR。已进化为 Agentic 评审：不只看 diff，会主动探索代码库相关文件，理解变更的全局影响。

可通过 Rulesets 设为仓库必需评审步骤，对 Coding Agent 产出的 PR 同样生效 —— AI 写的代码先过 AI 评审。

自动初审

PR 创建即触发，数分钟给出意见：Bug、性能、安全隐患、可读性

一键修复建议

评审意见附 Suggested Changes，可直接 commit 应用

自定义评审规范

用 copilot-instructions.md 定义团队关注点（如 SQL 参数化检查），持续降噪

评审 → 修复闭环

意见直接转给 Coding Agent 自动修复并更新 PR，形成 AI 审 → AI 改循环

边界：AI 评审不计入必需人工批准数 —— 无法判断「该不该做」，业务正确性与架构取舍仍由人终审。

Agent Merge — 自动清障与合并

补上「PR 已批准却没人收尾」的最后一公里

是什么

让 Agent 持续监控 PR 直到合并完成：自动解决合并冲突、修复 CI 失败、响应评审意见，所有门禁满足后自动执行合并。

「Fix with Copilot」一键委托冲突解决：云端分析双方变更意图，生成语义正确的方案并验证后推送。

六条设计原则

GitHub Agentic SDLC 的底层哲学 —— 自建体系也应遵循
01

PR 为中心

一切工作以 PR 呈现 —— 天然可评审、可讨论、可回滚，Agent 协作即状态流转

02

Draft-first

AI 产出默认是草稿 —— 信任需要赢得，而不是预设；人确认后才升级为正式变更

03

人是 Approver
不是 Executor

人可以不写代码，但不能被绕过 —— 终审权与门禁设定权不可让渡给 AI

04

最小权限

copilot/* 分支限制、出站防火墙、不可自批自合 —— Agent 能力越强，围栏越要硬

05

一切可审计

每一步推理、命令、变更都有 Session Log —— 出问题可回放，责任可追溯

06

上下文即资产

Instructions / AGENTS.md / Spaces 沉淀在仓库 —— 越用越准，成为组织的复利

治理不是给 AI 减速，而是让 AI 可以放心加速
